# 可视化监控

可视化监控（Memory Dashboard）是内置的 Web 界面，无需使用 CLI 即可浏览、搜索和管理记忆，实时直观地查看 Agent 的记忆状态。

## 访问地址

在浏览器中打开：

```
http://localhost:8230/ui
```

如果服务运行在远端服务器上，将 `localhost` 替换为服务器 IP 或域名，例如：

```
http://ec2-xx-xx-xx-xx.compute-1.amazonaws.com:8230/ui
```

## 功能说明

| 功能 | 说明 |
|---|---|
| **Agent 筛选** | 按 agent ID 过滤记忆列表（如 `dev`、`main`、`blog`）|
| **语义搜索** | 按语义含义搜索记忆，结果按相关性得分降序排列（最相关在前）|
| **列表视图** | 按 `created_at` 降序浏览所有记忆（最新在前）|
| **Auto 标签** | Pipeline 自动生成的记忆（`auto_digest`、`auto_dream` 等）显示绿色 **Auto** 标签 |
| **短期/长期标签** | 有 `run_id` 的记忆显示橙色**短期**徽章；无 `run_id` 的长期记忆显示紫色**长期**徽章 |
| **得分显示** | 搜索结果显示相关性得分百分比 |
| **展开/折叠** | 点击记忆卡片展开完整内容和元数据 |
| **删除** | 点击删除按钮移除单条记忆 |

## UI vs CLI 选择指南

| 场景 | 推荐方式 |
|---|---|
| 快速浏览 Agent 记住了什么 | ✅ 可视化监控 |
| 按主题搜索验证记忆质量 | ✅ 可视化监控 |
| 删除过时或错误的记忆 | ✅ 可视化监控 |
| 编程方式添加记忆 | CLI |
| 批量操作（按 run_id 过滤等）| CLI |
| 自动化 pipeline 集成 | CLI |

## 标签说明

| 标签 | 颜色 | 含义 |
|---|---|---|
| **Auto** | 绿色 | 由自动化 pipeline 写入的记忆（非手动写入）|
| **短期** | 橙色 | 短期记忆，有 `run_id`，7 天后自动升级或归档 |
| **长期** | 紫色 | 长期记忆，无 `run_id`，永久保留直到主动删除 |
| **Shared** | 蓝色 | 来自共享知识池的记忆（`category=experience` 或 `procedural`）|
